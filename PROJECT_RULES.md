# AutoCore AI — Project Rules & Context

## Project Overview
This project is the official landing page for AutoCore AI.

The website has already been designed and partially completed using Claude Design, then exported as a ZIP project and opened in Visual Studio Code.

The current project uses a static frontend structure (HTML/CSS/JS exported from Claude Design).

The objective now is NOT to redesign the website from scratch.

The objective is to:
- improve functionality,
- complete missing features,
- maintain visual consistency,
- optimize UX,
- and prepare the landing page for production.

---

# Main Priorities

Current priorities in order:

1. Finish demo videos
2. Embed YouTube demos into the website
3. Finish contact form functionality
4. Improve responsive behavior
5. Improve animations and UX polish
6. Optimize performance and consistency
7. Prepare deployment-ready version

---

# VERY IMPORTANT — Preserve Existing Design

The current design is already close to final.

DO NOT:
- change the color palette,
- change typography,
- redesign sections aggressively,
- replace layouts unnecessarily,
- remove premium styling,
- simplify the design,
- convert it into a generic corporate site.

The visual identity is already approved.

---

# Brand Style

AutoCore AI should feel:

- premium,
- modern,
- futuristic,
- cinematic,
- clean,
- technological,
- startup/SaaS inspired.
- trustworthy and approachable,
- friendly while remaining professional,
- emotionally connected to business owners,
- warm and human-centered communication.

Visual direction:
- dark premium UI,
- deep navy/blue backgrounds,
- yellow accent color,
- soft glows,
- minimalistic layouts,
- glassmorphism details,
- smooth animations.

---

# Communication Style

Website copy should:
- be simple,
- easy for non-technical business owners to understand,
- focused on business results,
- avoid excessive technical jargon.

Prioritize:
- more clients,
- automation benefits,
- saving time,
- professionalism,
- growth.

Avoid overly technical wording.

# Preferred Workflow

When modifying the website:

1. Prefer adding new sections/components first
2. Compare visually before deleting old sections
3. Avoid destructive replacements
4. Maintain consistency with existing design
5. Prioritize preserving approved visuals

Never aggressively rewrite large parts of the website without necessity.

---

# Contact Form Status

The contact form currently exists visually but is NOT functional yet.

Current issue:
- users can submit visually,
- but no real message/email is received.

Future implementation needed:
- real form handling,
- email delivery,
- lead collection,
- submission confirmation.

Potential solutions may include:
- Formspree,
- EmailJS,
- Resend,
- backend integration,
- Google Sheets integration.

Keep implementation simple and reliable.

# Mobile & Responsive Priority

- tablet,
- mobile devices.

- oversized text,

- visually impressive but clean,
